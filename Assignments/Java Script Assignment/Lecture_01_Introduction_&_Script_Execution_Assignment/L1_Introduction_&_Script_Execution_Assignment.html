Lecture 01 – Introduction & Script Execution

🟢 Beginner Level (Foundations)

1. First Script Execution

Task:

  • Write a JavaScript file that logs three messages.
  • Observe execution order in the console.

Goal:

Understand top-to-bottom synchronous execution.

Answer:

1. First Script Execution

JavaScript runs line by line from top to bottom. This is called synchronous execution.

👉 That means:

  • First line runs first
  • Second line runs next
  • No skipping or jumping

💡 Think of it like reading a book 📖 — one line at a time

2. Inline vs External Script

Task:

  • Add one script inline in HTML and one external JS file.
  • Log messages from both.

Goal:

Learn how scripts are loaded and executed.

Answer:

2. Inline vs External Script

There are two ways to write JavaScript:

🔹 Inline Script

  • Written inside HTML file (<script> tag).
  • Executes immediately where it is placed.

🔹 External Script

  • Written in a separate .js file.
  • Loaded into HTML using <script src="">.

👉 Execution depends on where the script is placed, not just type.

3. Execution Order Prediction

Task:

  • Predict output before running:

console.log("One");

console.log("Two");

console.log("Three");

Goal:

Build execution reasoning instead of trial-and-error.

Answer:

3. Execution Order Prediction

JavaScript does not guess or rearrange code.

👉 It simply runs:

  • First statement → first output
  • Second statement → second output

console.log("One"); // Fist Output

console.log("Two"); // Second Output

console.log("Three"); // Third Output

💡 So output always follows the written order.

4. Blocking Script Experiment

Task:

  • Add a large for loop before a console.log.
  • Observe browser responsiveness.

Goal:

Understand blocking behavior.

Answer:

4. Blocking Script Behavior

JavaScript runs on a single thread (one task at a time).

👉 If you run a heavy task (like a big loop):

  • Browser becomes slow 😵
  • Page may freeze
  • Other actions must wait

💡 This is called blocking.

5. Script Placement Test

Task:

  • Place the script in <head> and then before </body>.
  • Compare page load behavior.

Goal:

Learn why script placement matters.

Answer:

5. Script Placement (Head vs Body)

🔹 Script in <head>

  • Runs before page loads
  • Can slow down page display

🔹 Script before </body>

  • Runs after page loads
  • Faster user experience 🚀

👉 Best practice: place scripts at the bottom.

6. Async vs Sync Comparison

Task:

  • Compare output of synchronous logs vs <setTimeout>.

Goal:

Understand deferred execution.

Answer:

6. Sync vs Async Execution

🔹 Synchronous

  • Runs step-by-step
  • Waits for each line

🔹 Asynchronous (like setTimeout)

  • Does not block
  • Runs later ⏳

💡 Example idea: "Do this later, continue now."
